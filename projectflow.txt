# When we (add) it,there will be a warning:
#warning: in the working copy of 'experiments/mynotebook.ipynb', LF will be replaced by CRLF the next time Git touches it

This warning is simply about how different operating systems handle invisible line endings (the characters your computer uses every time you press Enter to start a new line of text or code).

CRLF (Carriage Return Line Feed): Used natively by Windows. It uses two hidden characters (\r\n) to mark the end of a line.

LF (Line Feed): Used natively by Mac and Linux (and often by interactive data science tools like Jupyter Notebooks/.ipynb files). It uses only a single hidden character (\n) to mark the end of a line.

Why Git is telling you this:
Your Jupyter Notebook file was originally saved using Linux-style line endings (LF). Because you are working on a Windows system,
 Git is warning you that it is automatically converting those invisible line endings to Windows-style (CRLF) so that your local applications
can read and modify the file smoothly without formatting glitches.

params.yaml setup:
  1>.import yaml
  2>.add func:
    
def load_params(params_path: str) ->dict:
  """load parameters from a YAML file ."""
    try:
      with open(params_path, 'r') as file:
        params = yaml.safe_load(file)
      logger.debug('Parameters retrieved from %s', params_path)
      return params
    except FileNotFoundError:
      logger.error('File not found: %s', params_path)
        raise
    except  yaml.YAMLError as e:
      logger.error('YAML error: %s', e) 
      raise
    except Exception as e:
      logger.error('Unexpected error: %s', e)
      raise
